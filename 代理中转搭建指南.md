# 云服务器代理中转桥接（Proxy Bridge）搭建指南

本教程旨在解决以下问题：如何将复杂的加密节点（如 VLESS、VMess、Trojan）转换成项目可识别的标准 HTTP 代理，并部署在云服务器上以支持 OpenAI 自动化注册。

---

## 一、 方案原理
由于本项目后端使用标准的 HTTP 协议进行通信，无法直接解析私有加密协议。通过搭建中转桥接，我们可以实现流量转换：

`项目后端 (HTTP)` -> `本地中转 (sing-box)` -> `加密节点 (VLESS/TLS)` -> `目标站点 (OpenAI)`

---

## 二、 准备配置文件
在云服务器上创建一个目录（如 `/root/phantom/proxy/`），并新建 `config.json` 文件。

### 1. 配置文件模板 (`config.json`)
请将以下内容复制并修改：

```json
{
  "log": {
    "level": "info"
  },
  "inbounds": [
    {
      "type": "http",
      "tag": "http-in",
      "listen": "127.0.0.1",
      "listen_port": 2080
    }
  ],
  "outbounds": [
    {
      "type": "vless",
      "tag": "proxy",
      "server": "tw3.dexlos.com", 
      "server_port": 8080,
      "uuid": "你的UUID",
      "tls": {
        "enabled": true,
        "server_name": "tw3.dexlos.com",
        "utls": {
          "enabled": true,
          "fingerprint": "chrome"
        },
        "alpn": ["h3"]
      }
    },
    {
      "type": "direct",
      "tag": "direct"
    }
  ]
}
```
> [!NOTE]
> 请根据你获取到的节点信息，修改 `server`、`server_port`、`uuid`、`tls.server_name` 等关键字段。

## 三、 部署中转服务（推荐使用 Docker）

通过 Docker 可以快速拉起服务且无需复杂配置：

```bash
# 进入配置目录
cd /root/phantom/proxy/

# 启动 Docker 容器
docker run -d \
  --name phantom-proxy-bridge \
  --restart always \
  -v $(pwd)/config.json:/etc/sing-box/config.json \
  -p 127.0.0.1:2080:2080 \
  ghcr.io/sagernet/sing-box:latest \
  run -c /etc/sing-box/config.json
```

## 四、 项目配置与验证

### 1. 连通性测试
在服务器终端运行以下指令，确认中转是否工作正常：
```bash
curl -x http://127.0.0.1:2080 https://www.google.com -I
```
如果返回 `HTTP/2 200`，说明本地 HTTP 代理已就绪。

### 2. 在项目中使用
打开 PhantomDrop 控制面板，在工作流配置的 `proxy_url` 字段中填写：
`http://127.0.0.1:2080`

## 五、 常见问题与进阶建议

### 1. 关于 OpenAI 注册 (403 错误)
如果你在使用此代理后依然收到 `403 Forbidden`：
*   **原因**：你使用的节点 IP 属于“机房/数据中心”，被 OpenAI 深度屏蔽。
*   **对策**：需要更换为**“住宅代理 (Residential Proxy)”**。住宅代理商通常直接提供用户名密码认证的 HTTP 链接，无需搭建此中转。

### 2. 账号分发
如果注册成功，建议在工作流参数中配置 `cpa_url` 和 `cpa_key`，实现注册成功后自动将账号信息推送到你的分发平台。

### 3. 多节点切换
如果你有多个节点，可以修改 `config.json` 中的 `outbounds` 列表，并使用 `selector` 类型进行筛选，或为不同的工作流开启不同的本地端口。
