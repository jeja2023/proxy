## [0.0.6] - 2026-05-12

### 🔐 代理鉴权连接失效修复
- **旧代理连接强制失效**：修复开启或修改 `SINGBOX_HTTP_USER` / `SINGBOX_HTTP_PASS` 后，面板热重载配置虽然写入了代理鉴权，但 sing-box 内核中既有 HTTP/HTTPS 代理长连接仍可继续复用，导致“不带用户名和密码也能继续使用”的问题。现在面板会在检测到代理鉴权启用、关闭或账号密码变化后，热重载配置并主动断开所有旧代理连接，确保新鉴权策略立即生效。
- **升级启动兜底清理**：当面板容器升级或重启后，如果当前 HTTP 代理鉴权已启用，面板会在启动阶段主动通知 sing-box 断开历史代理连接，覆盖“升级后旧网站连接仍然可用”的场景。
- **重连行为明确**：历史连接被清理后，客户端若自动重连但未配置代理用户名和密码，将被新鉴权策略拒绝；只有配置正确用户名和密码的客户端才能重新建立代理连接并继续使用。
- **热重载密钥兼容**：面板触发 sing-box Clash API 热重载与断开连接时，会兼容当前密钥和本次提交的新密钥，降低更新 `CLASH_API_SECRET` 时热重载失败的概率。
- **回归测试补齐**：新增代理鉴权变更判定单元测试，覆盖启用鉴权、修改密码、关闭鉴权与鉴权未变化不误断连接等场景。

## [0.0.5] - 2026-05-11

### 🔐 代理鉴权与热加载支持
- **热重载机制修复**：修复了在 `.env` 中配置代理用户名与密码 (`SINGBOX_HTTP_USER` / `SINGBOX_HTTP_PASS`) 后，仅重启容器无法热加载到底层内核 `config.json` 从而导致密码鉴权不生效的问题。现在通过面板保存、重构或终端脚本重构后即可立刻热加载并强制开启密码鉴权。
- **命令行工具环境加载**：优化了 `scripts/generate_singbox.py` 脚本，使其在宿主机终端单独执行时（例如通过 `sudo`），能够自动检索并加载项目根目录下的 `.env` 环境变量，避免由于 `sudo` 环境变量擦除导致 `VAULT_PASSWORD` 缺失而报错。

### 🪟 节点库管理与弹窗交互修复
- **查看节点列表 JavaScript 报错修复**：修复了在节点库管理中，点击“查看”节点由于弹窗元素 ID 变更不一致（`view-vault-name` 与 `view-vault-content` 错位）而导致页面 JavaScript 抛出 `Cannot set properties of null (setting 'textContent')` 崩溃的问题。
- **导入选择弹窗 4 种导入方式还原**：全面还原了之前意外丢失的 4 种节点导入方式（「订阅链接」「二维码」「手动粘贴」「文件导入」）的带图标按钮布局；并显式设置了按钮文字颜色，修复了由于缺少 `choice-label` 样式类而在某些浏览器下白底白字无法看清的问题。
- **订阅导入目标库名空白修复**：修复了订阅导入弹窗与选择导入弹窗中，由于目标库名元素 ID 重名（同为 `import-target-name`）产生 HTML ID 冲突，导致订阅导入弹窗标题中的目标节点库名称无法正常显示的问题（通过重构为 `import-target-name-sub` 并双向绑定成功修复）。

- **节点卡片宽度与密度优化**：多轮调整节点卡片网格宽度、内边距、字号和间距，使节点列表更宽、更紧凑，长节点名更不容易换行。

### 🧪 代理质量检测报告
- **新增质量检测入口**：节点选择页新增「检测常用网站」按钮，点击后打开「代理质量检测报告」弹窗。
- **弹窗式检测流程**：检测面板不再直接占用节点选择页空间；弹窗内支持「一键检测」全部项目，也支持对每个检测项单独检测。
- **质量报告摘要**：报告展示节点名、通过/告警/失败/挑战/待检测计数、质量分、等级、出口 IP、出口地区、基础延迟和检测时间。
- **摘要区域压缩优化**：出口 IP、出口地区、基础延迟、检测时间改为同一行展示，并在摘要中加入当前节点面板信息（地区、类型、协议、倍率、链路、健康分）。
- **出口 IP 与地区识别增强**：出口信息不再只依赖单一服务，改为依次尝试 `ipapi`、`ipinfo`、`ipwho.is` 与 Cloudflare trace，提高识别成功率。
- **出口地区中文化**：常见国家/地区代码和英文国家名统一映射为中文显示，如「香港」「美国」「日本」「台湾」「新加坡」等。
- **质量评分算法调整**：评分按已测项目比例扣分，避免新增检测项后单个失败项过度拉低质量分。
- **挑战状态说明纳入检测体系**：报告支持区分「失败」与「挑战」，挑战用于表示目标站可达但触发验证码、风控或浏览器验证等情况。

- **新增常用站点小卡**：Google、YouTube、Netflix、GitHub、OpenAI、Cloudflare、Wikipedia、Microsoft 以横向小卡展示检测结果，不再占用表格行。
- **常用网站单行布局**：在宽弹窗下 8 个站点固定同一行显示，缩小图标、卡片宽度和间距，避免出现水平滚动条；窄屏下自动切换为 4 列。
- **状态文案明确化**：解锁类站点显示「已解锁 / 未解锁 / 检测失败」，普通可用性站点显示「可访问 / 访问受限 / 检测失败」，不再在小卡中显示 HTTP 状态码。
- **延迟独立展示**：常用网站小卡将延迟单独靠右显示，避免被状态文本或 HTTP 文本截断。
- **本地图标体系**：常用站点图标全部改为本地 inline SVG 或本地 CSS 图形，不再请求任何在线 logo / favicon 资源。
- **品牌图标多轮优化**：优化 Google、YouTube、Netflix、GitHub、OpenAI、Cloudflare、Wikipedia、Microsoft 图标比例、配色和小尺寸识别度；OpenAI 改为白底单色结形标志，Cloudflare 改为更接近官方橙色云标。
- **解锁与可访问语义区分**：「可访问」表示站点连通；「已解锁」表示服务型/地区限制站点未检测到明显地区限制或不可用提示。

### 🔐 鉴权与 Auth 检测
- **面板鉴权 / CSRF 检测**：质量报告中明确显示当前检测请求是否已通过面板登录态与 CSRF 校验。
- **Clash API Auth 检测**：检测面板访问 sing-box Clash API 的鉴权状态，返回状态码、延迟和说明。
- **HTTP 代理鉴权检测**：检测 HTTP 代理入口是否启用用户名密码，以及当前检测请求是否能通过代理鉴权。
- **未启用鉴权不再误判失败**：当 HTTP 代理入口未配置用户名密码时，结果显示「无需鉴权」并计为通过，可用性由基础连通性和站点检测判断。

### 🩺 检测稳定性与结果解释优化
- **基础连通性多目标兜底**：基础连通性不再只依赖单个 `gstatic` 目标，改为尝试 Google 204、Cloudflare trace、GitHub、Microsoft 任意一个可达即视为基础连通正常。

- **报告表格分层**：常用网站/解锁结果使用小卡展示，基础连通性、鉴权、Auth、AI API 等诊断项保留表格展示。

## [0.0.3] - 2026-05-07

### 服务器部署与 HTTPS 代理入口补充
- **修复服务器 Docker 代理端口映射问题**：将 Docker Compose 中的 HTTP 代理端口改为“公网宿主机端口 -> 容器内固定 2080”，避免 `.env` 中自定义 `SINGBOX_HTTP_PORT` 后，容器内部没有对应监听端口而导致 `ERR_PROXY_CONNECTION_FAILED`。
- **新增 HTTPS 正向代理入口**：新增 `SINGBOX_HTTPS_PROXY_ENABLED`、`SINGBOX_HTTPS_PROXY_PUBLIC_PORT`、`SINGBOX_TLS_CERT_PATH`、`SINGBOX_TLS_KEY_PATH`、`SINGBOX_TLS_SERVER_NAME` 等配置项，可生成带 TLS 的 `https-in` 入站，使客户端能够通过 `https://域名:端口` 加密连接 NetHub 代理入口。
- **新增证书挂载与端口暴露配置**：Docker Compose 增加 `/certs` 只读证书挂载，并预留 `SINGBOX_HTTPS_PROXY_PUBLIC_PORT -> 2443` 映射，方便服务器直接挂载 Let’s Encrypt 或现有证书文件。
- **增强服务器部署诊断工具**：重写 `scripts/diagnose_server.py`，支持检查 HTTP/HTTPS 代理入站、Docker 端口映射、TLS 证书路径、面板端口、代理认证状态与防火墙/安全组放行建议，并避免输出敏感密钥明文。
- **中文化真实环境配置文件**：将真实 `.env` 和 `.env.example` 的说明注释统一改为中文，并修复原文件中部分变量被写入注释同行导致无法被程序读取的问题，确保 `CLASH_API_SECRET`、`PANEL_SESSION_SECRET`、`PANEL_PORT`、`SINGBOX_IMAGE`、`SINGBOX_HTTP_PORT` 等配置均为有效 `KEY=value` 行。
- **补充 HTTPS 入口测试覆盖**：新增 HTTPS 代理入站生成测试与 TLS 证书路径缺失报错测试，当前测试套件扩展至 8 项并通过验证。

### 🛡️ 安全加固与生产部署基线 (Security & Production Readiness)
- **全站变更请求 CSRF 防护**：面板新增会话级 CSRF Token，下发于 `/api/meta`，前端统一在所有 POST / PUT / DELETE 请求中携带 `X-CSRF-Token`。除登录接口外，所有会修改节点库、连接、选择器或配置的接口均完成校验覆盖。
- **订阅与测速 URL SSRF 防护升级**：统一校验 HTTP/HTTPS 外部 URL，拒绝 localhost、回环、内网、链路本地、保留地址等目标；订阅拉取改为手动跟随重定向，并对每一跳目标重新校验，阻断“合法入口跳转内网”的风险。
- **前端动态内容安全渲染**：连接监控等来自内核/网络的数据不再通过动态 `innerHTML` 拼接渲染，改为 DOM + `textContent` 构建，降低 XSS 与异常 HTML 注入风险。
- **Docker 构建上下文保护**：新增 `.dockerignore`，默认排除 `.env`、`config.json`、`data/`、`bin/`、`.venv/`、本地日志等敏感或庞大文件，避免容器构建时误带入私密数据。
- **生产依赖锁定**：新增 `requirements.lock`，Dockerfile 改为使用锁定依赖构建，提升生产环境可复现性；`requirements.txt` 继续保留为开发依赖入口。
- **默认密钥风险收口**：Docker Compose 不再提供固定的 `PANEL_SESSION_SECRET` 默认值，改由环境变量或面板运行时持久化随机密钥处理，减少生产误用弱默认密钥的风险。

### 📡 订阅源维护与节点库工作流增强 (Subscription & Vault Workflow)
- **导入自动去重**：节点解析和配置构建链路新增顺序保留去重能力，重复分享链接会被统计并剔除，避免 selector 出现重复出口。
- **订阅来源持久记录**：通过订阅导入的节点库会记录来源 URL、来源类型、最近导入时间、唯一节点数与重复节点数，为后续维护提供可靠元数据。
- **一键刷新订阅库**：节点库列表中，具备订阅来源的库自动显示“刷新”操作。刷新会复用已保存来源重新拉取、解码、去重、写入 Vault，并热重构运行配置。
- **可选后台自动刷新**：新增 `PANEL_SUB_REFRESH_INTERVAL_MIN`。当同时配置 `VAULT_PASSWORD` 时，面板可按固定间隔后台刷新所有记录过来源的订阅库，并写入审计日志。
- **导入变更预览**：手动导入弹窗新增“预览变更”，保存前可查看新增、移除、保留与重复节点数量，降低误覆盖节点库的风险。
- **节点名片段保留修复**：修复分享链接解析时 `#节点名` 片段被截断的问题，导入后的节点名称保真度更高。

### 📈 节点健康评分与运维可观测性 (Health & Observability)
- **节点健康评分体系**：新增 JSON 持久化健康仓库 `data/node_health.json`，测速结果会沉淀为成功率、平均延迟、最近错误、最近观测时间与 0-100 健康分。
- **节点卡片健康展示**：节点选择卡片新增健康分展示，帮助快速识别延迟波动、失败率较高或尚未观测的出口节点。
- **云端网关总览**：状态页新增“云端网关总览”卡片，聚合运行时长、节点总数、节点库启用状态、订阅库数量、健康均分、异常节点数与后台刷新状态。
- **健康状态 API**：新增 `/api/node-health`，供面板或后续自动策略读取当前节点健康快照。
- **网关摘要 API**：新增 `/api/gateway-summary`，用于集中查询云端网关运行态、配置更新时间、Vault 规模与后台订阅刷新结果。
- **自动切换策略基础增强**：现有故障自切继续兼容，测速结果同步写入健康评分，为后续按健康分排序、择优切换和失败率策略奠定基础。

### ⚙️ 配置生成与导出稳定性 (Config & Export Reliability)
