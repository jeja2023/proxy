# 通用代理中转服务使用说明

本工具用于将加密协议（如 VLESS）桥接为标准 HTTP 代理。

## 文件列表

1.  **`config.json`**: sing-box 核心配置文件。
2.  **`docker-compose.yml`**: Docker Compose 编排文件。
3.  **`部署脚本.sh`**: 适用于 Linux 服务器的一键部署脚本。
4.  **`启动服务.ps1`**: 适用于 Windows 环境的一键启动脚本。

## 实施步骤

### 1. 修改节点配置
打开 `config.json`，修改 `outbounds` 部分的节点信息：

### 2. 启动服务

#### 在 Linux 云服务器上 (支持非 root 用户)：
```bash
# 赋予权限
chmod +x 部署脚本.sh
# 运行
./部署脚本.sh
```

> [!TIP]
> 如果你使用的是非 root 账户（如 `ubuntu`），脚本会将文件存放在你的家目录 `~/proxy-bridge` 下。

#### 在 Windows 本地：
右键点击 `启动服务.ps1`，选择“使用 PowerShell 运行”。

### 3. 使用方法
启动成功后，代理地址为：
`http://127.0.0.1:2080`
或者使用云服务器的公网 IP 访问（需开启 2080 端口）：
`http://你的服务器IP:2080`

## 注意事项
- **安全性**: 默认监听 `0.0.0.0:2080`。如果在公网服务器运行，请务必在安全组中限制来源 IP，防止代理被盗用。
- **协议支持**: 目前模板为 VLESS 协议。如需 VMess 或 Trojan，请参考 sing-box 官方文档修改 `config.json`。
